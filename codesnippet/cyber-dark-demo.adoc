== Why WebClient?

The evolution of HTTP clients in Spring

=== RestTemplate vs WebClient

[.columns]
=== RestTemplate vs WebClient

[.column.code-before%step]
--
.RestTemplate (blocking)
[source,java]
----
// Blocks the thread until response
RestTemplate rest = new RestTemplate();
User user = rest.getForObject(
    "/api/users/{id}", User.class, 42);

// Thread sits idle during I/O
// Limited scalability under load
----

[.text-danger]
Blocking I/O — one thread per request
--

[.column.code-after%step]
--
.WebClient (reactive)
[source,java]
----
// Non-blocking, returns immediately
WebClient client = WebClient.create();
Mono<User> user = client.get()
    .uri("/api/users/{id}", 42)
    .retrieve()
    .bodyToMono(User.class);

// Thread is free for other work
----

[.text-accent]
Non-blocking — thousands of concurrent requests

== Core Patterns

Building blocks for HTTP calls

=== GET Request

[source,java]
----
@Service
public class UserClient {
    private final WebClient webClient;

    UserClient(WebClient.Builder builder) {
        this.webClient = builder
            .baseUrl("https://api.example.com")
            .defaultHeader(ACCEPT, APPLICATION_JSON_VALUE)
            .build();
    }

    Mono<User> findById(Long id) {
        return webClient.get()
            .uri("/users/{id}", id)
            .retrieve()
            .bodyToMono(User.class);
    }
}
----

=== POST with Body

[source,java]
----
Mono<User> create(CreateUserRequest request) {
    return webClient.post()
        .uri("/users")
        .contentType(MediaType.APPLICATION_JSON)
        .bodyValue(request)
        .retrieve()
        .bodyToMono(User.class);
}
----

[.key-concept]
`bodyValue()` for simple objects, `body()` with `BodyInserters` for streams

=== WebClient Methods at a Glance

[cols="1,1", options="header"]
|===
| Method | Description

| `get()` / `post()` / `put()` / `delete()`
| Start building a request with the given HTTP method

| `.uri()`
| Set the request URI with optional path variables

| `.header()` / `.contentType()`
| Add request headers or set content type

| `.bodyValue()` / `.body()`
| Set the request body (object or reactive stream)

| `.retrieve()`
| Execute the request and access the response

| `.bodyToMono()` / `.bodyToFlux()`
| Decode response body to a single value or a stream

| `.onStatus()`
| Handle specific HTTP status codes

| `.exchangeToMono()`
| Full access to the `ClientResponse` for advanced use
|===

== Error Handling

Graceful failures in reactive pipelines

=== Status Code Handling

[source,java]
----
Mono<User> findById(Long id) {
    return webClient.get()
        .uri("/users/{id}", id)
        .retrieve()
        .onStatus(HttpStatusCode::is4xxClientError, resp ->
            resp.bodyToMono(ErrorResponse.class)
                .map(e -> new UserNotFoundException(e.message())))
        .onStatus(HttpStatusCode::is5xxServerError, resp ->
            Mono.error(new ServiceUnavailableException()))
        .bodyToMono(User.class);
}
----

=== Retry & Timeout

[source,java]
----
Mono<User> findWithResilience(Long id) {
    return webClient.get()
        .uri("/users/{id}", id)
        .retrieve()
        .bodyToMono(User.class)
        .timeout(Duration.ofSeconds(3))
        .retryWhen(Retry.backoff(3, Duration.ofMillis(500))
            .filter(ex -> ex instanceof WebClientResponseException.ServiceUnavailable))
        .onErrorResume(TimeoutException.class,
            ex -> Mono.error(new ServiceTimeoutException()));
}
----

== Testing

Verifiable mock servers for WebClient

=== MockWebServer

[source,java]
----
@ExtendWith(MockitoExtension.class)
class UserClientTest {
    static MockWebServer server;
    UserClient client;

    @BeforeAll
    static void setup() throws Exception {
        server = new MockWebServer();
        server.start();
    }

    @BeforeEach
    void init() {
        var webClient = WebClient.builder()
            .baseUrl(server.url("/").toString()).build();
        client = new UserClient(webClient);
    }

    @Test
    void shouldFetchUser() {
        server.enqueue(new MockResponse()
            .setBody("{\"name\":\"Alice\"}")
            .setHeader(CONTENT_TYPE, APPLICATION_JSON_VALUE));

        StepVerifier.create(client.findById(1L))
            .expectNextMatches(u -> u.name().equals("Alice"))
            .verifyComplete();
    }
}
----

== Summary

Wrapping up

=== Key Takeaways

* [.text-accent]*WebClient* replaces RestTemplate for non-blocking HTTP
* [.text-accent]*Mono/Flux* compose naturally with reactive pipelines
* [.text-accent]*onStatus* for granular error handling per status code
* [.text-accent]*Retry + Timeout* for production resilience
* [.text-accent]*MockWebServer* for deterministic integration tests

[.key-concept]
Embrace reactive: fewer threads, more throughput, +
better resource utilization under load.
