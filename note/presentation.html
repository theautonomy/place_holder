Setup

Configuring WebClient

WebClientConfig.java 
@Configuration
public class WebClientConfig {

    @Bean
    public WebClient webClient() {
        HttpClient httpClient = HttpClient.create()
            .option(ChannelOption.CONNECT_TIMEOUT_MILLIS, 5000)
            .doOnConnected(conn ->
                conn.addHandlerLast(new ReadTimeoutHandler(5000, MILLISECONDS))
                    .addHandlerLast(new WriteTimeoutHandler(5000, MILLISECONDS)));

        return WebClient.builder()
            .baseUrl("https://jsonplaceholder.typicode.com")
            .clientConnector(new ReactorClientHttpConnector(httpClient))
            .filter(logRequest())
            .filter(errorHandlingFilter())
            .build();
    }
}

Fetching Data

Single item (Mono) 
webClient
    .get()
    .uri("/posts/{id}", id)
    .accept(APPLICATION_JSON)
    .retrieve()
    .bodyToMono(Post.class)
    .timeout(Duration.ofSeconds(5));
Collection (Flux) 
webClient
    .get()
    .uri("/posts")
    .accept(APPLICATION_JSON)
    .retrieve()
    .bodyToFlux(Post.class)
    .retryWhen(Retry.backoff(3, Duration.ofSeconds(2)))
    .timeout(Duration.ofSeconds(10));

Mono<T> = 0 or 1 item   |   Flux<T> = 0 to N items

    .retrieve().bodyToMono(Post.class);

// UPDATE
webClient.put().uri("/posts/{id}", id)
    .contentType(APPLICATION_JSON)
    .bodyValue(post)
    .retrieve().bodyToMono(Post.class);

// DELETE
webClient.delete().uri("/posts/{id}", id)
    .retrieve().bodyToMono(Void.class);
Error Handling

Handling Failures Gracefully

WebClientService.java 
webClient.get().uri("/posts/{id}", id)
    .retrieve()
    .onStatus(s -> s.equals(HttpStatus.NOT_FOUND),
        resp -> Mono.error(new RuntimeException("Not found")))
    .onStatus(HttpStatusCode::is4xxClientError,
        resp -> Mono.error(new RuntimeException("Client error")))
    .onStatus(HttpStatusCode::is5xxServerError,
        resp -> Mono.error(new RuntimeException("Server error")))
    .bodyToMono(Post.class)
    .retryWhen(Retry.backoff(3, Duration.ofSeconds(1))
        .filter(ex -> !(ex instanceof NotFound)))
    .onErrorResume(ex -> Mono.empty());

onStatus per-status  →  retryWhen with filter  →  onErrorResume fallback

Reactive Composition

Sequential Chaining

getUserById(userId)

    getPostsByUserId(userId)
      .collectList()

        user.name() + " has "

Concurrent Requests

Flux.fromIterable(userIds)
  .flatMap(
    this::getUserById,
    3  // concurrency level
  )
  .timeout(
    Duration.ofSeconds(10)
  );

flatMap chains async calls   |   Second arg controls parallelism

Spring 6+

Declarative HTTP Interface

JsonPlaceholderClient.java 
public interface JsonPlaceholderClient {

    @GetExchange("/posts")
    Flux<Post> getAllPosts();

    @GetExchange("/posts/{id}")
    Mono<Post> getPostById(@PathVariable Long id);

    @PostExchange("/posts")
    Mono<Post> createPost(@RequestBody Post post);

    @DeleteExchange("/posts/{id}")
    Mono<Void> deletePost(@PathVariable Long id);
}

Define an interface → Spring generates the WebClient implementation via proxy

Models

Java Records as DTOs

Post.java 
public record Post(
    @JsonProperty("id") Long id,
    @JsonProperty("userId") Long userId,
    @JsonProperty("title") String title,
    @JsonProperty("body") String body
) {
    // Convenience constructor
    public Post(Long userId,
               String title,
               String body) {
        this(null, userId, title, body);
    }
}
  • 1 Immutable by default — no setters needed
  • 2 Auto-generated equals(), hashCode(), toString()
  • 3 Compact syntax replaces verbose POJOs
  • 4 Works seamlessly with Jackson for JSON serialization
